## ADJACENT NON-OPERATIONAL NEIGHBORS
- upstream: LEXICON — Typed lexical knowledge substrate (primary lexical basis for L03 lexical uptake when provided)
- downstream: none

## UPSTREAM
- direct upstream phases:
  - L02 — Morphosyntax candidate space
- optional non-phase substrate:
  - LEXICON — typed lexical knowledge query output; when provided it is the primary lexical basis before any heuristic fallback
- допустимо опираться на:
  - typed outputs direct upstream phases и их declared status/failure markers
  - typed lexical substrate query artifacts as weak lexical evidence only
  - load-bearing telemetry direct upstream phases в рамках их claim boundaries
- запрещено предполагать:
  - данные/гарантии непрямых фаз как будто они уже доступны на этом шве
  - молчаливое заполнение пробелов upstream без unknown/abstain/reject

## DOWNSTREAM
- direct downstream phases:
  - L04 — Dictum candidate construction
- downstream должен получить:
  - typed phase output с явным status/bounds для operational use
  - seam telemetry, достаточную для post-hoc реконструкции seam-решения
  - inspectable lexical/reference ambiguity + syntax-instability markers, пригодные для candidate-level dictum construction
- downstream не должен получать:
  - неявные truth-claims или policy decisions вне роли фазы
  - hidden side effects / обход F01 canonical write seam

## OPEN SEAMS TO LEAVE
- логика direct downstream фаз остаётся на их стороне шва
- дальние (не direct) зависимости и кросс-этажные оптимизации не закрываются здесь

## FORBIDDEN OVERREACH
- подмена соседних фаз (upstream/downstream) собственной ad-hoc логикой
- утверждение доменных фактов/решений, требующих ещё не реализованных фаз

## SEAM OBLIGATIONS
- принимать и возвращать только typed seam artifacts
- при недостатке оснований возвращать explicit unknown/abstain/reject вместо форсинга
- lexicon-first lexical basis:
  - when typed lexicon input is provided, L03 lexical basis must be derived from lexicon query artifacts first
  - heuristic lexical path is allowed only as explicit fallback after lexicon no-usable/capped/unknown outcome
- lexicon handoff discipline:
  - contour entry without lexicon handoff must not look like normal lexical mode
  - missing lexicon handoff is emitted as explicit bounded degradation:
    - `lexicon_handoff_missing`
    - `lexical_basis_degraded`
    - `no_strong_lexical_claim_without_lexicon`
  - gate restrictions must expose this degraded mode (no silent normalization)
- fallback honesty:
  - fallback usage must be inspectable (`heuristic_fallback_used`, reason, no-strong-claim marker)
  - fallback cannot erase lexicon unknown/partial/conflict/capped states
  - fallback cannot silently look equivalent to lexicon-backed lexical knowledge
- preserve basis distinction through persistence/snapshot:
  - `lexicon_backed` vs `lexicon_capped_unknown` vs `heuristic_fallback` vs `no_usable_lexical_basis`

## LOAD-BEARING SEAM TELEMETRY
- входной artifact id/source refs и upstream lineage на шве
- seam decision/status + reason + imposed downstream restrictions
- lexical basis mode counts and fallback reasons:
  - lexicon-backed mention count
  - lexicon-capped unknown mention count
  - heuristic fallback mention count
  - no-usable lexical basis mention count
  - no-strong-lexical-claim-from-fallback marker
- degraded-handoff markers:
  - `lexicon_handoff_missing`
  - `lexical_basis_degraded`
  - `no_strong_lexical_claim_without_lexicon`

## REQUIRED SEAM TESTS
- valid seam pass-through от direct upstream с корректным typed output
- invalid/insufficient upstream contract -> honest unknown/abstain/reject
- boundary test: фаза не выполняет работу direct downstream фаз
- lexicon integration tests:
  - lexicon-backed basis dominates heuristic lexical guess when lexicon provides usable support
  - fallback is explicit and capped when lexicon is unknown/partial/capped
  - missing lexicon handoff enters explicit degraded bounded mode and is reflected by gate restrictions
  - lexicon ablation measurably degrades lexical grounding basis behavior
  - persist/reconstruct/continue preserves lexical basis provenance (lexicon-backed vs fallback)

## SEAM FALSIFIERS
- фаза silently промотирует слабый/неполный upstream в сильный downstream claim
- downstream получает только raw payload без required seam status/constraints
- L03 silently bypasses available lexicon basis and emits heuristic lexical guesses as normal grounding
- fallback used but not inspectable in typed output/telemetry/snapshot
- lexicon unknown/capped state erased by fallback so output appears cleanly resolved
- contour path without lexicon handoff is silently treated as normal lexical mode
