## UPSTREAM

  - C05 — Temporal validity / selective revalidation
- допустимо опираться на:
  - typed outputs direct upstream phases и их declared status/failure markers

  - данные/гарантии непрямых фаз как будто они уже доступны на этом шве

## DOWNSTREAM
- direct downstream phases:
  - D01 — Causal observatory monitor
- downstream должен получить:
  - typed phase output с явным status/bounds для operational use
  - seam telemetry, достаточную для post-hoc реконструкции seam-решения
- downstream не должен получать:
  - неявные truth-claims или policy decisions вне роли фазы
  - hidden side effects / обход F01 canonical write seam

## OPEN SEAMS TO LEAVE
- логика фазы D01 должна оставаться на её стороне шва
- дальние (не direct) зависимости и кросс-этажные оптимизации не закрываются здесь

## FORBIDDEN OVERREACH
- RT01 не выбирает mode semantics вместо C04.
- RT01 не переинтерпретирует validity semantics вместо C05.
- RT01 не становится planner/orchestrator authority.

## SEAM OBLIGATIONS
- принимать и возвращать только typed seam artifacts
- при недостатке оснований возвращать explicit unknown/abstain/reject вместо форсинга
- исполнять контрактный порядок и downstream gates, не подменяя authority соседних фаз

## RELATION SEMANTIC CONTRACT
- C01 -> RT01: `requires` (runtime contour requires typed continuity carrier)
- C04 -> RT01: `arbitrates` (RT01 executes C04-selected mode contract, не переопределяя mode смысл)
- C05 -> RT01: `gates` (C05 legality can block/restrict continue path in runtime contour)
- RT01 -> D01: `observes_only` (runtime contour exposes trace surface for observability without semantic control)

## LOAD-BEARING SEAM TELEMETRY
- входной artifact id/source refs и upstream lineage на шве
- seam decision/status + reason + imposed downstream restrictions

## REQUIRED SEAM TESTS
- valid seam pass-through от direct upstream с корректным typed output
- invalid/insufficient upstream contract -> honest unknown/abstain/reject
- boundary test: RT01 не выполняет semantic работу C04/C05

## SEAM FALSIFIERS
- RT01 silently переопределяет mode/validity authority соседних фаз
- downstream получает только raw payload без required seam status/constraints
