  - неявные truth-claims или policy decisions вне роли фазы
  - hidden side effects / обход F01 canonical write seam

## OPEN SEAMS TO LEAVE
- логика фаз C06, O04, P01, P04... должна оставаться на их стороне шва
- дальние (не direct) зависимости и кросс-этажные оптимизации не закрываются здесь

## FORBIDDEN OVERREACH
- подмена соседних фаз (upstream/downstream) собственной ad-hoc логикой
- утверждение доменных фактов/решений, требующих ещё не реализованных фаз

## SEAM OBLIGATIONS
- принимать и возвращать только typed seam artifacts
- при недостатке оснований возвращать explicit unknown/abstain/reject вместо форсинга

## LOAD-BEARING SEAM TELEMETRY
- входной artifact id/source refs и upstream lineage на шве
- seam decision/status + reason + imposed downstream restrictions

## REQUIRED SEAM TESTS
- valid seam pass-through от direct upstream с корректным typed output
- invalid/insufficient upstream contract -> honest unknown/abstain/reject
- boundary test: фаза не выполняет работу direct downstream фаз

## SEAM FALSIFIERS
- фаза silently промотирует слабый/неполный upstream в сильный downstream claim
- downstream получает только raw payload без required seam status/constraints
