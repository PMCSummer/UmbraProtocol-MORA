## Canonical Seams
- `compute_viability_control_state(..., boundary_spec=None, calibration_spec=None) -> ViabilityControlResult`
- `evaluate_viability_downstream_gate(...) -> ViabilityGateDecision`
- `persist_viability_control_result_via_f01(...) -> execute_transition(...)`

## UPSTREAM CONTRACT
- typed-only critical path:
  - `RegulationState | RegulationResult` (R01)
  - `AffordanceResult` (R02)
  - `PreferenceState | PreferenceUpdateResult` (R03)
- raw/untyped payloads on critical path are rejected.
- R04 may consume upstream lineage/telemetry for traceability only within authority bounds.

## DOWNSTREAM CONTRACT
- downstream receives typed viability-control state + directives + restrictions.
- downstream must not treat R04 output as final action selection or as world truth.
- downstream must preserve uncertainty/restriction markers.

## SEAM OBLIGATIONS
- keep viability pressure separate from preference-over-means:
  - threat magnitude is boundary/worsening/persistence driven
  - preference contributes only recoverability hints
  - preference epistemic blockages must not be counted as failed restorations in pressure/persistence math
- calibration must be typed, versioned, and inspectable:
  - no hidden threshold/weight drift in code path
  - incompatible calibration/boundary/preference contracts trigger explicit compatibility markers and strong-override capping
  - calibration remains explicit rule-based config, not adaptive self-tuning
- emit explicit:
  - pressure level
  - escalation stage
  - override scope
  - persistence state
  - de-escalation conditions
  - uncertainty/no-strong-override markers
- recoverability must be factorized (not a single opaque proxy):
  - viable affordance coverage
  - restorative capacity evidence
  - blocked/unavailable fraction
  - preference support bias
  - evidence quality
  - recent failed restoration penalty
- preserve honest fallback states:
  - insufficient_observability
  - boundary_uncertain
  - mixed_deterioration
  - unresolved_conflict
  - degraded_mode_only
  - no_strong_override_claim
- mixed deterioration must cap strong override unless clean single-axis threat attribution basis is present.
- incompatible prior viability state (calibration/formula mismatch) must not be reused as normal persistence/de-escalation basis.
- do not perform planner/policy selection.
- runtime mutation only via F01 canonical write seam.

## PHASE ROLE CONTRACT
- authority_role: `gating`
- computational_role: `evaluator`
- R04 computes viability pressure and has gated survival-constraining authority.
- R04 does not become planner/executor and does not replace downstream semantics.

## RELATION SEMANTIC CONTRACT
- R04 -> C01/C02/C03/C05: `modulates` (viability pressure reshapes allowable stance without direct semantic rewrite)
- R04 -> A01/A02/M01/O02: `gates` (downstream permissiveness is conditionally constrained by viability gate outcome)
- R04 -> C04/S01/S02/S03/S04/S05: `overrides_survival` (critical viability pressure has privileged pre-emption surface)

- no narrative-survival, no emotional-metaphysical claims.

- uncertainty paths cannot suppress strong override claims.
- calibration incompatibility is silently ignored.
