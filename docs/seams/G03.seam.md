## OPERATIONAL UPSTREAM
- G02 — Frame runtime / semantic graph compilation

## OPERATIONAL DOWNSTREAM
- G04 — Discourse provenance / perspective chaining

## CONCEPTUAL DEPENDENCIES
- upstream context (lineage-preserved, not runtime-direct intake): G01, L05
- intended downstream chain beyond direct runtime consumer: G05, G06, G07, T01, T02

## NORMATIVE CONTOUR / INTENDED DOWNSTREAM CHAIN
- runtime segment: `G02 -> G03 -> G04`
- intended continuation: `G04 -> G05 -> G06 -> G07` (and then downstream consumers)

## ADJACENT NON-OPERATIONAL NEIGHBORS
- upstream:
  - G04 — Discourse provenance / perspective chaining [refines/causal]
  - G04 — Discourse provenance / perspective chaining [blocks_claim/validation]
  - G07 — Targeted clarification / uncertainty intervention [blocks_claim/validation]
- downstream: none

## UPSTREAM
- direct upstream phases:
  - G02 — Frame runtime / semantic graph compilation
- допустимо опираться на:
  - typed outputs direct upstream phases и их declared status/failure markers
  - load-bearing telemetry direct upstream phases в рамках их claim boundaries
- запрещено предполагать:
  - данные/гарантии непрямых фаз как будто они уже доступны на этом шве
  - молчаливое заполнение пробелов upstream без unknown/abstain/reject
  - что presence `G01/L05` lineage fields эквивалентен runtime-direct intake этих фаз

## DOWNSTREAM
- direct downstream phases:
  - G04 — Discourse provenance / perspective chaining
- downstream должен получить:
  - typed phase output с явным status/bounds для operational use
  - seam telemetry, достаточную для post-hoc реконструкции seam-решения
- downstream не должен получать:
  - неявные truth-claims или policy decisions вне роли фазы
  - hidden side effects / обход F01 canonical write seam

## OPEN SEAMS TO LEAVE
- логика фаз G04, G05, G06, G07... должна оставаться на их стороне шва
- дальние (не direct) зависимости и кросс-этажные оптимизации не закрываются здесь

## FORBIDDEN OVERREACH
- подмена соседних фаз (upstream/downstream) собственной ad-hoc логикой
- утверждение доменных фактов/решений, требующих ещё не реализованных фаз

## EXPLICIT NON-CLAIMS
- G03 не потребляет `G01` и `L05` как runtime-direct inputs в текущем build signature
- G03 не выполняет работу G04/G05/G06/G07/T01/T02
- G03 не делает discourse provenance chaining, acquisition closure, framing или intervention planning

## SEAM OBLIGATIONS
- принимать и возвращать только typed seam artifacts
- при недостатке оснований возвращать explicit unknown/abstain/reject вместо форсинга

## LOAD-BEARING SEAM TELEMETRY
- входной artifact id/source refs и upstream lineage на шве
- seam decision/status + reason + imposed downstream restrictions

## REQUIRED SEAM TESTS
- valid seam pass-through от direct upstream с корректным typed output
- invalid/insufficient upstream contract -> honest unknown/abstain/reject
- boundary test: фаза не выполняет работу direct downstream фаз

## SEAM FALSIFIERS
- фаза silently промотирует слабый/неполный upstream в сильный downstream claim
- downstream получает только raw payload без required seam status/constraints
