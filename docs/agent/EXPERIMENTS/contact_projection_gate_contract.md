# CONTACT-PROJECTION-GATE Contract Demo Notes

## Purpose
Demonstrate safe projection from closed UMWELT0 contact frames into existing AB/ACP/AP lineage-compatible subject inputs.

## Demo cases
- valid symbolic contact
- mixed multichannel contact
- action surface projected as basis only
- knowledge hint preserved as hint (not truth)
- language claim preserved as testimony hint (not truth)
- sensory candidate preserved as candidate (not object truth)
- request-correlated effect lineage
- passive event effect lineage
- hidden eval blocked
- scenario label blocked
- blocked UMWELT0 frame => projection noop/blocked
- oversized contact bounded

## Accepted projection pattern
- AB receives public observations/effects/residue/uncertainty/conflict.
- ACP01 receives action/capability/context/hint/constraint basis refs.
- AP01 lineage receives correlation refs only.

## Blocked projection pattern
- UMWELT0 blocked/rejected frames.
- action-policy surfaces.
- hidden/scenario basis.
- unsafe truth-oracle markers in effects.

## Authority boundary
Projection output always keeps:
- `action_request_emitted=False`
- `world_submission_emitted=False`
- `fact_claimed=False`
- `cause_confirmed=False`
- `value_assigned=False`
- `mature_recipe_claimed=False`
- `mature_skill_claimed=False`
- `automation_claimed=False`

## Non-claim boundary
This gate is compatibility glue only. It is not:
- autonomous symbolic progression,
- world runner,
- UMWELT-S compiler,
- planner/selector/executor.
